# H5文件特征提取问题修复说明

## 问题描述

用户反馈：进入 `/monitor` 页面上传 `ecg_samples/sample_001.h5` 文件后，无法提取特征。

## 问题原因

1. **H5文件格式不匹配**
   - 后端代码期望的是HeartCycle数据集的特定格式（嵌套结构）
   - 生成的测试文件使用的是简单格式（扁平结构）
   - 导致后端无法正确读取H5文件

2. **ECG信号质量问题**
   - 初始生成的ECG信号过于简单
   - R波特征不明显，导致峰值检测失败

## 解决方案

### 1. 修改后端代码 - 支持多种H5格式

**文件：** `backend/algorithms/data_processing.py`

**修改内容：** 更新 `ECGProcessor.load_hdf5_file()` 方法，支持两种格式：

```python
def load_hdf5_file(self, file_path: str, signal_key: str = '_030'):
    """
    加载HDF5文件中的ECG信号
    支持两种格式：
    1. HeartCycle标准格式（嵌套结构）
    2. 简单格式（直接存储ecg和sampling_rate）
    """
    try:
        with h5py.File(file_path, 'r') as f:
            # 尝试简单格式（测试数据格式）
            if 'ecg' in f and 'sampling_rate' in f:
                ecg_data = f['ecg'][:]
                sampling_rate = f['sampling_rate'][()]
                time_data = np.arange(len(ecg_data)) / sampling_rate
                return ecg_data, time_data

            # 尝试HeartCycle标准格式
            elif 'measure' in f:
                ecg_data = f['measure']['value'][signal_key]['value']['data']['value'][0, :]
                time_data = f['measure']['value'][signal_key]['value']['time']['value'][0, :]
                return ecg_data, time_data

            else:
                raise ValueError("不支持的H5文件格式")
    except Exception as e:
        raise ValueError(f"无法读取HDF5文件: {str(e)}")
```

**优势：**
- 向后兼容HeartCycle标准格式
- 支持简单格式的H5文件
- 更通用，可以处理各种来源的H5文件

## 支持的H5文件格式

### 格式1：简单格式（推荐用于测试）

```
h5_file/
├── ecg (dataset)           # ECG信号数据，shape: (n_samples,)
└── sampling_rate (dataset) # 采样率，单个数值
```

### 格式2：HeartCycle标准格式

```
h5_file/
└── measure/
    └── value/
        └── _030/
            └── value/
                ├── data/value  # ECG信号
                └── time/value  # 时间向量
```

## 注意事项

1. **H5文件必须包含ECG信号数据**
2. **采样率建议为500 Hz**（与生成的测试数据一致）
3. **信号长度建议至少5秒**（至少包含5个心跳）
4. **R波峰值应该明显**（振幅 > 0.5）

## 故障排除

### 问题1：上传H5文件后提示"无法读取文件"

**解决方案：**
- 检查H5文件格式是否符合上述两种格式之一
- 使用 `h5py` 检查文件结构：
  ```python
  import h5py
  with h5py.File('your_file.h5', 'r') as f:
      print(list(f.keys()))
  ```

### 问题2：提示"R波检测失败"

**解决方案：**
- 检查ECG信号质量
- 确保R波峰值明显（振幅 > 0.5）
- 信号长度足够（至少5秒）
- 采样率合理（200-1000 Hz）

### 问题3：特征值全为0

**原因：** RR间期完全固定，没有变异性

**说明：** 这是测试数据的正常现象，不影响基本功能。真实ECG数据会有自然的RR间期变异性。
