# 用户管理功能修复说明

## 问题描述

1. **进入用户管理界面需要重新登录** - 令牌过期导致
2. **禁用用户后无法重新启用** - 缺少启用功能
3. **无法重置用户密码** - 缺少重置密码功能

### 2. 后端修改

#### 文件：`backend/app/api/v1/auth.py`

**新增API端点：**

1. **启用用户**
   - 路由：`PUT /api/v1/auth/users/{user_id}/activate`
   - 权限：仅管理员
   - 功能：将用户状态设置为启用

2. **重置密码**
   - 路由：`POST /api/v1/auth/users/{user_id}/reset-password`
   - 权限：仅管理员
   - 功能：重置用户密码为 `password123`
   - 返回：新密码

#### 文件：`backend/app/services/auth_service.py`

**新增服务方法：**

```python
def activate_user(self, user_id: int) -> bool:
    """启用用户"""
    user = self.get_user_by_id(user_id)
    if not user:
        raise UserNotFoundError("用户不存在")

    user.is_active = True
    self.db.commit()
    return True

def reset_user_password(self, user_id: int, new_password: str = "password123") -> str:
    """重置用户密码"""
    user = self.get_user_by_id(user_id)
    if not user:
        raise UserNotFoundError("用户不存在")

    user.hashed_password = self.get_password_hash(new_password)
    self.db.commit()
    return new_password
```

### 3. JWT令牌修复（已完成）

**问题：** 令牌有效期太短（30分钟）导致频繁需要重新登录

**解决：**
- access_token有效期：30分钟 → 8小时
- refresh_token有效期：7天 → 30天
- 添加自动令牌刷新机制

详见：[JWT令牌过期修复说明.md](./JWT令牌过期修复说明.md)

## 安全考虑

1. **权限控制**
   - 所有操作都需要admin权限
   - 不能禁用自己
   - 不能操作admin账号（防止误操作）

2. **审计日志**
   - 所有操作都会记录到审计日志
   - 包含操作者、操作类型、目标用户、IP地址等

3. **密码安全**
   - 重置后的密码使用pbkdf2_sha256加密存储
   - 建议用户首次登录后立即修改密码

## 常见问题

### Q1: 为什么还是需要重新登录？
**A:** 需要重启后端服务，并确保JWT令牌修复已生效（access_token有效期8小时）。

### Q2: 启用按钮不显示？
**A:** 只有已禁用的用户才会显示启用按钮。检查用户状态列是否显示"已禁用"。

### Q3: 不能操作admin用户？
**A:** 这是安全设计，防止误操作导致系统无管理员。

### Q4: 重置密码后用户无法登录？
**A:** 确保使用新密码 `password123` 登录。建议用户登录后立即修改密码。

### Q5: 操作后列表没有更新？
**A:** 操作成功后会自动刷新列表。如果没有更新，手动刷新页面（F5）。
