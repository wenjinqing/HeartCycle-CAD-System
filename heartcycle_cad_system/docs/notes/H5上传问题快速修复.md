# 🔧 H5文件上传问题 - 快速修复指南

## 问题现象

上传H5文件时提示：
```
操作失败: 无法读取HDF5文件 D:\Graduate Work\heartcycle_cad_system\data\raw\sample_001.h5:
"Unable to synchronously open object (object 'measure' doesn't exist)"
```

## 原因

后端代码已修复，但**后端服务需要重启**才能加载新代码。

## 解决方案（3步）

### 步骤1：停止后端服务

在运行后端的终端窗口按 `Ctrl + C` 停止服务。

### 步骤2：重启后端服务

```bash
cd "D:\Graduate Work\heartcycle_cad_system"
python scripts/start_backend.py
```

或者直接运行：
```bash
cd "D:\Graduate Work\heartcycle_cad_system\backend"
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 步骤3：测试上传

1. 刷新前端页面（F5）
2. 进入"风险监测"页面
3. 上传 `测试数据/ecg_samples/sample_001.h5`
4. 填写患者信息
5. 点击"开始分析"

应该能够成功提取特征！

## 技术说明

### 修改的文件

1. `backend/algorithms/data_processing.py`
2. `backend/app/algorithms/data_processing.py`

### 修改内容

`load_hdf5_file()` 方法现在支持两种H5格式：
- **简单格式**：`ecg` + `sampling_rate`（测试数据使用）
- **HeartCycle格式**：`measure/value/...`（原始数据集）

### 为什么需要重启

Python的 `import` 机制会缓存已加载的模块。修改代码后，必须重启Python进程才能加载新代码。

使用 `--reload` 参数启动uvicorn可以自动检测代码变化并重启，但有时需要手动重启。
