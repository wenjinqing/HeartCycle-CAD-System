## 系统概述

HeartCycle 是一个基于机器学习的冠心病风险预测系统，提供数据分析、模型训练、患者管理等功能。

**技术栈：** Vue 3 + Element Plus + **Vue CLI 5**（`@vue/cli-service`）

**访问地址：** http://localhost:8080（`npm run dev` / `npm run serve`，端口以终端输出为准）

## 登录与权限

### 用户角色

系统支持4种用户角色，不同角色拥有不同权限：

| 角色 | 权限说明 |
|------|---------|
| **admin** | 管理员，拥有所有权限 |
| **doctor** | 医生，可管理患者、查看报告 |
| **researcher** | 研究员，可训练模型、进行实验 |
| **patient** | 患者，仅可查看自己的数据 |

### 登录页面

**路由：** `/login`

**测试账号：** 见 `测试数据/test_users.json`

**操作步骤：**
1. 输入用户名和密码
2. 点击"登录"按钮
3. 登录成功后自动跳转到首页

### 2. 风险监测 (Monitor)

**路由：** `/monitor`

**功能：** 实时心电数据分析和风险评估

**操作步骤：**

#### 方式1：仅使用临床特征
1. 填写患者基本信息（年龄、性别、BMI等）
2. 点击"开始分析"
3. 查看风险评估结果

#### 方式2：上传心电数据
1. 填写患者基本信息
2. 点击"上传HDF5文件"
3. 选择H5格式的心电数据文件
4. 点击"开始分析"
5. 系统自动提取HRV特征并进行综合评估

**测试文件：** `测试数据/ecg_samples/` 目录下的 `.h5` 文件

### 3. H5文件快速训练 (TrainH5Auto)

**路由：** `/train-h5-auto`

**权限：** admin, doctor, researcher

**功能：** 上传H5文件自动识别标签并训练模型

**操作步骤：**

#### 步骤1：上传H5文件
1. 拖拽或点击选择多个H5文件
2. 系统会自动在H5文件目录查找 `SubjectMetadata.csv`
3. 从元数据中提取 `Disease_Status` 字段作为标签
4. 点击"下一步：配置参数"

#### 步骤2：配置参数
- **模型类型：** 选择 LR/SVM/RF/XGBoost/LightGBM/Stacking/Voting
- **交叉验证折数：** 2-10折，默认5折
- **高级选项：**
  - 使用SMOTE处理数据不平衡
  - 自动超参数优化（耗时较长）
- **特征提取选项：**
  - 使用已有R波标注
  - 提取HRV特征
  - 提取临床特征
- 点击"下一步：确认训练"

#### 步骤3：开始训练
1. 确认配置信息
2. 点击"开始训练"
3. 等待训练完成（显示进度条）
4. 查看训练结果和标签统计

**测试文件：**
- H5文件：`测试数据/h5_training_samples/` 目录
- 元数据：`测试数据/h5_training_samples/SubjectMetadata.csv`

**注意事项：**
- H5文件必须与 `SubjectMetadata.csv` 在同一目录
- 元数据文件必须包含 `Disease_Status` 列
- 建议至少上传10个H5文件以获得较好的训练效果

---

### 4. 深度学习模型训练 (TrainDeepLearning)

**路由：** `/train-deep-learning`

**权限：** admin, doctor, researcher

**功能：** 训练深度学习模型（CNN/LSTM/GRU/CNN-LSTM）

**操作步骤：**

1. **选择H5文件：** 至少选择10个文件
2. **选择模型类型：**
   - **1D-CNN：** 适合提取局部特征，速度快
   - **LSTM：** 适合长序列依赖，内存占用大
   - **GRU：** LSTM的简化版，速度更快
   - **CNN-LSTM：** 结合两者优势，性能最佳
3. **配置参数：**
   - 信号长度：1000-10000，默认5000
   - 训练轮数：10-200，建议50-100
   - 批次大小：8-256，建议32
   - 学习率：0.0001-0.01，建议0.001
   - 测试集比例：10%-40%
   - 验证集比例：10%-30%
4. **模型校准：** 启用Platt Scaling或Isotonic Regression
5. 点击"开始训练"
6. 查看已训练的模型列表

**测试文件：** `测试数据/h5_training_samples/` 目录下的 `.h5` 文件

### 5. H5格式转换 (H5Converter)

**路由：** `/h5-converter`

### 6. 批量预测 (BatchPredict)

**路由：** `/batch-predict`

**权限：** admin、doctor、researcher（与路由 `meta.roles` 一致；**patient** 不可访问）

**功能：** 批量上传患者数据进行风险预测

**操作步骤：**

1. 下载CSV模板
2. 按模板格式填写患者数据
3. 上传填写好的CSV文件
4. 选择预测模型
5. 点击"开始预测"
6. 查看预测结果
7. 导出结果报告

**测试文件：** `测试数据/batch_predict_sample.csv`

**CSV格式要求：**
```csv
age,gender,bmi,systolic_bp,diastolic_bp,cholesterol,glucose,smoking,family_history
45,male,28.5,130,85,220,105,1,0
```

### 7. 患者管理 (PatientList)

**路由：** `/patients`

**权限：** admin, doctor

**功能：** 管理患者信息

**操作步骤：**

#### 添加患者
1. 点击"添加患者"按钮
2. 填写患者信息：
   - 基本信息：姓名、性别、年龄、出生日期
   - 联系方式：手机号、地址
   - 紧急联系人信息
   - 病史、过敏史、备注
3. 点击"确定"保存

#### 搜索患者
- 在搜索框输入姓名、患者编号或手机号
- 按回车或点击搜索按钮

#### 查看/编辑/删除
- 点击"查看"查看患者详情
- 点击"编辑"修改患者信息
- 点击"删除"删除患者（需确认）

**测试文件：** `测试数据/test_patients.json`

### 10. 论文实验系统 (ThesisExperiment)

**路由：** `/thesis-experiment`

**权限：** admin, researcher

**功能：** 完整的论文实验流程

**实验步骤：**

#### 步骤1：数据预处理
1. 上传CSV数据集（必须包含 `CAD_risk` 列）
2. 配置参数：
   - KNN邻居数：3-10
   - 异常比例：0.01-0.1
   - 训练集比例：0.5-0.8
3. 点击"开始预处理"
4. 查看预处理结果：
   - 训练集/验证集/测试集样本数
   - 缺失值处理、异常检测、标准化、类别平衡状态

#### 步骤2：特征工程
1. 配置参数：
   - 方差阈值：0.001-0.1
   - 互信息百分比：0.5-1.0
   - RFE特征数：20-50
2. 点击"开始特征工程"
3. 查看特征选择结果

#### 步骤3：模型训练
1. 点击"开始训练模型"
2. 系统自动训练RF、XGBoost、LightGBM
3. 查看最佳模型和AUC

#### 步骤4：结果分析
- **性能对比：** 查看各模型的准确率、灵敏度、特异性、AUC
- **SHAP分析：** 查看特征重要性和临床解读
- **下载报告：** 导出完整实验报告

**测试文件：** `测试数据/experiment_dataset.csv`

**注意事项：**
- 数据集必须包含 `CAD_risk` 列作为标签
- 建议样本数 ≥ 1000 以获得稳定结果
- 实验过程不可逆，建议先备份数据

## 常见问题

### 1. 上传文件失败
- 检查文件格式是否正确
- 检查文件大小是否超限
- 检查网络连接

### 2. 训练任务卡住
- 查看浏览器控制台错误信息
- 检查后端服务是否正常运行
- 刷新页面重试

### 3. 权限不足
- 确认当前用户角色
- 联系管理员分配权限

### 4. 数据格式错误
- 参考测试数据文件格式
- 检查必填字段是否完整
- 确保数据类型正确
