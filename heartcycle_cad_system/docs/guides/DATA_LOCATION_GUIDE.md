## 📝 论文写作数据引用

### 第5.1节 数据集介绍

```markdown
本研究使用的冠心病风险预测数据集共包含10,000例患者样本，
其中阳性样本（确诊冠心病）3,000例，占比30%；阴性样本
（未患病）7,000例，占比70%。数据集涵盖42个临床特征变量，
包括人口统计学特征、生理指标、实验室检查结果以及心率变异性
（HRV）特征。

数据集按照7:1.5:1.5的比例划分为训练集（7,000例）、验证集
（1,500例）和测试集（1,500例）。
```

**数据来源**: `data/cad_dataset_10k.csv`

### 第5.2节 数据预处理

```markdown
数据预处理流程如下：

1. 缺失值处理：采用K近邻（KNN）插补法，设置k=5。
   原始数据无缺失值。

2. 异常值检测：使用孤立森林算法，设置异常样本比例为5%。
   检测到500个异常样本，采用边界值替换策略处理。

3. 数据标准化：采用Z-score标准化，标准化后均值接近0，
   标准差接近1。

4. 类别平衡：使用SMOTE过采样，将训练集正负样本比例从
   7:3调整为1:1，生成2,800个合成样本。
```

**数据来源**: 运行日志输出

### 第5.3节 特征工程

```markdown
采用多阶段特征选择策略：

第一阶段：方差阈值法（阈值=0.01）剔除4个低方差特征，
保留38个特征。

第二阶段：互信息法保留前80%的特征，筛选出30个特征。

第三阶段：RFE方法选择30个核心特征。

特征扩展：构建14个临床交互特征，最终特征维度为43维。
```

**数据来源**: 运行日志输出

### 第5.4节 模型性能对比

```markdown
表5-1展示了各模型在测试集上的性能对比。

| 模型 | 准确率 | 灵敏度 | 特异性 | AUC |
|------|--------|--------|--------|-----|
| Logistic Regression | 87.2% | 86.2% | 87.6% | 0.935 |
| Random Forest | 83.7% | 74.0% | 87.9% | 0.891 |
| XGBoost | 85.5% | 73.8% | 90.6% | 0.907 |
| LightGBM | 84.6% | 74.0% | 89.1% | 0.905 |

从表中可以看出，Logistic Regression模型表现最优，准确率达到87.2%，
AUC为0.935。该模型在灵敏度和特异性之间取得了较好的平衡，灵敏度
为86.2%，特异性为87.6%。XGBoost模型虽然准确率略低（85.5%），
但特异性最高（90.6%），在降低误诊率方面表现突出。
```

**数据来源**: `results/model_comparison.csv`

### 第5.5节 SHAP分析

```markdown
采用SHAP方法对最佳模型进行可解释性分析。全局特征重要性
分析显示，运动频率是最重要的预测因子（重要性占比7.21%），
其次是RMSSD（6.20%）和收缩压（5.80%）。

生活方式因素的高重要性验证了改善生活习惯对降低冠心病风险
的重要作用。HRV特征（RMSSD、SDNN、SD1）合计贡献约16%的
预测能力，表明心率变异性是评估心血管健康的重要指标。

传统危险因素如血压（收缩压5.80%、舒张压4.60%）、血糖
（5.57%）、血脂（LDL 5.17%、总胆固醇4.54%）也显示出
较高的预测价值，与临床经验一致。
```

**数据来源**: `results/shap_summary.png` + 运行日志

## 📞 常见问题

**Q: 实验结果在哪里？**
A: 在 `results/` 目录下，包含CSV、PNG、TXT文件。

**Q: 如何重新运行实验？**
A: 运行 `python backend/run_experiment.py`

**Q: 数据集可以修改吗？**
A: 可以，修改 `backend/algorithms/dataset_generator.py` 后重新生成。

**Q: 图表不清晰怎么办？**
A: PNG文件默认300 DPI，可以直接用于论文。如需更高分辨率，
   修改代码中的 `dpi=300` 参数。
